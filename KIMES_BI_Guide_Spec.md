## 0. 이 문서의 사용법 (Claude Code에게)

이 명세서는 위에서 아래로 순서대로 작업하면 된다. 각 섹션은 다음 구조를 따른다.

- **목적**: 왜 이 페이지/섹션이 존재하는가
- **사용자 시나리오**: 참가업체가 어떤 상황에서 이걸 보는가
- **콘텐츠 명세**: 페이지에 들어갈 모든 텍스트·표·UI 요소
- **인터랙션**: 버튼·복사·다운로드·필터 등의 동작
- **금지사항**: 절대 들어가면 안 되는 것

작업 순서:
1. 먼저 §1, §2 (삭제 대상, 새 IA)를 읽고 현재 코드베이스에서 들어낼 것부터 정리
2. 그 다음 §3~§13의 페이지별 명세에 따라 새 페이지를 빌드
3. 마지막으로 §14 (디자인 시스템 토큰), §15 (반응형·접근성), §16 (배포 체크리스트) 적용

## 1. 작업의 본질 — 이걸 먼저 이해하지 않으면 다 망친다

### 1.1 가이드의 진짜 목적

이 가이드는 **"KIMES 브랜드를 외부가 함부로 쓰지 못하게 통제하면서, 참가업체에게는 충분히 활용할 수 있도록 권한을 명확히 정의하는 법적·실무적 매뉴얼"** 이다.

디자인 시스템이 아니다. 디자인 가이드도 절반의 정의일 뿐이다.

전시 BI 가이드의 본질은 **권리 관계 문서**다. 참가업체가 자기 마케팅에 KIMES 자산을 쓸 때, 어디까지 허용되고 어디부터는 승인이 필요하며, 어떤 행위는 절대 금지인지 — 이걸 흐릿하게 두면 1,400개 참가업체가 1,400개 다른 해석을 한다.

### 1.2 사용자

**유일한 사용자**: KIMES 2027 참가업체와 그 대행사·협력사.

세부적으로는:
- 참가업체 마케팅 담당자 (보도자료·SNS·홈페이지)
- 참가업체 디자인 팀 (부스·인쇄물·굿즈)
- 외주 부스 디자인 회사
- PR 에이전시
- 영상·SNS 콘텐츠 제작사
- (특별관 참가업체) 특별관 BI까지 추가로 따라야 함

**비대상**: 일반 관람객, 언론사, KIMES 사무국 내부, 디자인 시스템 운영팀.

### 1.3 4대 원칙 (모든 작업의 판단 기준)

1. **번역투 박멸** — 영어를 직역한 표현 전부 자연스러운 한국어로. ("프라이머리/세컨더리/베리언트/컨텍스트/처리/표면" 같은 음차·직역 금지)
2. **KIMES 단일 메인** — KIMES만 메인 브랜드. MedicomteK·BEAUTY&DERMA SEOUL·INSPIRE는 "특별관(Special Zone)"으로 정의. 4-브랜드 평행 구조 폐기.
3. **참가업체 외 자료 전부 삭제** — 사이니지·부스 마커·등록 데스크·모션 키트·BI 감사·내부 컴포넌트 시스템 전부 제거
4. **권리 관계 명시** — "할 수 있다 / 승인 필요 / 절대 금지" 3단계가 모든 행위에 명확히 라벨링되어야 함

| `12 — 간격 & 그리드` | 디자인 시스템 내부용 |
| `15 — 모션 & 애니메이션` (전체) | 워드마크 정적 사용만 허용. 모션은 사무국 영역 |
| `16 — 접근성 매트릭스` (WCAG 풀 매트릭스) | 참가업체용 가이드 범위 밖. 다만 §10 "디지털 자산"에 핵심 3줄만 남김 |

| `Applications — Print collateral` (프로그램북·카탈로그·ID뱃지·브로셔) | 전부 사무국이 만드는 인쇄물 |

### 2.2 표현·용어 단위 삭제 (전역 검색·치환)

| 삭제 대상 | 처리 |
|---|---|
| "슬래시컷" / "Slash-cut" / "슬래시컷 패밀리" | 전부 삭제. i 쐐기는 §3 로고 페이지에서 "워드마크의 디테일"로만 한 번 언급 |
| "프라이머리 / 세컨더리" | "메인 / 보조"로 치환 |
| "베리언트" | "변형" 또는 "버전" |
| "컨텍스트" | "상황" 또는 "쓰임새" |
| "토큰" (사용자 노출) | 내부 코드에서만 사용. 본문에서는 "값" 또는 항목명 |
| "Independent identity" | 전부 삭제 |
| "Pattern A/B/C" | 실제 사용 케이스명으로 치환 (예: "부스 헤더 표기", "보도자료 표기") |
| "~을/를 가집니다" (have 직역) | "~이 있습니다" 또는 능동형 |
| "~ 처리입니다 / ~ 표면입니다" | 자연스러운 동사형으로 |
| "본 시스템", "본 가이드" | "이 가이드" |
| "디자인 시스템" (이 사이트 자칭) | "BI 가이드" |

- 헤드라인 (Display, Pretendard 900, 1줄):
  > KIMES 2027 참가업체를 위한 브랜드 가이드

- CTA 2개 (좌→우):
  - **로고 다운로드** — KIMES Red 채움 (큰 버튼, 48px)
  - **가이드 보기** — 아웃라인 (큰 버튼, 48px)
- 우상단 메타 박스:
  - `2027.3.18 ─ 21 · COEX · 서울`

## 5. `/overview` — KIMES 한눈에 보기

### 5.1 목적

참가업체가 보도자료·홈페이지·SNS·이메일에 그대로 **복사해서 쓸 수 있는 공식 문구**를 한 페이지에 다 모은다. 이 페이지가 진짜 "참가업체용 가이드"의 시작점이다.

### 5.2 사용자 시나리오

- "보도자료 쓰는데 KIMES 정식 영문명이 뭐였지?"
- "회사 홈페이지에 KIMES 참가 알림 띄우는데 한 줄 소개가 필요해"
- "사장님 영문 인사말에 KIMES 참가 사실을 어떻게 표기하지?"

### 5.3 섹션 구성

#### 5.3.1 공식 명칭

표 형식. 각 행 우측에 `[복사]` 버튼. 클립보드 복사 + 토스트 ("복사되었습니다").

| 구분 | 한국어 | 영문 |
|---|---|---|
| 정식 | 제42회 국제 의료기기·병원설비 전시회 KIMES 2027 | KIMES 2027 — Korea International Medical & Hospital Equipment Show |
| 줄임 | KIMES 2027 | KIMES 2027 |
| 약칭 | KIMES | KIMES |

#### 5.3.2 일정·장소

| 구분 | 한국어 | 영문 |
|---|---|---|
| 기간 | 2027년 3월 18일(목) ~ 21일(일) | March 18–21, 2027 |
| 장소 | 코엑스 (서울 강남구 영동대로 513) | COEX, Seoul |
| 홀 | 1층 Hall A · 3층 Hall B–E | Hall A, B, C, D, E |
| 주최 | 한국이앤엑스 · KMDA · KMDIA | KOREA E&EX INC. with KMDA and KMDIA |
| 주관 | KIMES 사무국 | KIMES Secretariat |

각 행에 `[복사]` 버튼.

#### 5.3.3 한 줄 소개 (보일러플레이트)

길이별로 4개 버전. 보도자료 끝에 들어가는 표준 회사 소개문(boilerplate)처럼 쓸 수 있게.

**한국어**
- **40자 이내** — SNS·이메일 제목용
  > 한국 최대 의료기기 전시회 KIMES 2027, 3월 18~21일 코엑스
- **100자 이내** — 보도자료 리드문용
  > KIMES 2027은 1,400여 개 기업과 8만여 명이 참여하는 한국 최대 의료기기 전시회로, 3월 18~21일 코엑스에서 열립니다.
- **200자 이내** — 회사 소개 페이지·블로그용
  > KIMES는 한국이앤엑스가 KMDA·KMDIA와 함께 1980년부터 매년 개최해 온 대한민국 대표 의료기기·병원설비 전시회입니다. 2027년에는 제42회를 맞아 3월 18~21일 코엑스에서 열리며, 60여 개국 1,400여 개 기업이 참가해 영상진단·수술·치료·재활·뷰티·디지털 헬스 등 14개 카테고리를 아우릅니다.
- **400자 이내** — 보도자료 회사 소개 칸용

**영문**
- 동일하게 4개 길이.

## 6. `/logo` — 로고

### 6.1 목적

KIMES 워드마크를 정확하게 다운로드하고, 어떤 상황에서 어떤 버전을 써야 하는지 알려준다. 변형·재타이핑·임의 색상 변경을 명확히 막는다.

### 6.2 섹션 구성

#### 6.2.1 KIMES 워드마크

- 큰 이미지 1장 (워드마크 단독, 흰 배경)
- 짧은 설명 (3~4줄):
  > KIMES 워드마크는 1980년 첫 행사 이래 단단하고 직선적인 산세리프로 한국 의료기기 산업의 무게감을 표현해왔습니다. 강렬한 빨강과 두꺼운 글자, 그리고 i 안에 자리 잡은 검은 쐐기 — 이 세 요소가 KIMES만의 시각적 정체성입니다.

  ※ "슬래시컷 패밀리" 같은 표현은 절대 쓰지 않는다. i 쐐기는 "워드마크의 디테일"로만 언급.

#### 6.2.2 로고 버전 (그리드 카드)

4개 카드. 각 카드: 미리보기 + 라벨 + 다운로드 버튼군.

| 버전 | 용도 | 파일 |

| 그레이 | 본문 안 작은 표기·각주 | SVG · PNG · AI · EPS |

**연도 일체형 워드마크 (별도 제공)**

행사 직전 발표용 연도 일체형 워드마크(예: KIMES 2027 일체형)는 워드마크

정확합니다. 사무국 디자이너가 회차마다 별도 제공하므로 가이드 카드에는
두지 않습니다. 필요하면 사무국 brand 문의로 요청해주세요.

> 코드 합성(워드마크 + 텍스트 결합) 방식은 자간·시각 무게가 정확히 맞지
> 않아 시각 어색함이 발생하므로 사용하지 않습니다.

큰 다이어그램 1개.

#### 6.2.5 배경별 사용 — KT 11칸 격자(2개) + 브랜드 컬러 칩

KT BI 가이드의 11칸 명도 격자 시스템을 KIMES 톤으로 차용. 시각 도구
3개로 단순화 = 격자 2개(엄격한 사용 룰) + 단색 칩 1세트(4브랜드 컬러
케이스). Gray 워드마크는 자유 사용 → 한 줄 안내로 대체. 일반 컬러
그라디언트 격자는 4브랜드 칩 케이스와 정보 중복이라 제거.

KT 시스템과 같은 결의 핵심:

- Black/White 워드마크는 한 격자에서 묶어 처리 (KT가 같은 방식)
- 11칸 단색 격자로 명도 변화를 단계 시각화 (그라디언트가 아닌 명확한
  단계)
- 사용 X 구간은 빨간 대각선 빗금으로 시각 차단

**lede**

> 배경 명도와 컬러에 따라 워드마크를 선택합니다.

**(1) Primary Wordmark (Red) — 명도 배경색 규정**

명도 11칸 격자 (0% 흰 → 100% 검정, 10% 단위). 0–10% 칸(2칸)에만 KIMES
Red 워드마크 표시. 20–100% 칸(9칸)은 빨간 대각선 빗금. Primary 워드마크
는 대비를 위해 흰색·매우 옅은 배경에 한정.

> 룰: 명도 10% 이하 흰색·매우 옅은 배경에서만 사용. 그 외엔 Black 또는
> White로.

**(2) Black / White Wordmark — 명도 배경색 규정 ★ 묶음**

같은 11칸 명도 격자. 0–30% 칸(4칸)에 Black 워드마크, 40–100% 칸(7칸)에
White 워드마크. 빗금 없음 — 모든 명도에 둘 중 하나가 사용 가능.

> 룰: 0–30% 배경: Black 워드마크 / 40–100% 배경: White 워드마크.

**(3) Brand Color Backgrounds — KIMES 4브랜드 단색**

H3 sub-section "Brand Color Backgrounds (브랜드 컬러 배경)" 아래에 칩
5개를 가로 1줄 균등 분할 + 16px gap으로 분리. 각 칩 안에 권장 워드마크
표시.

| 구간 | 컬러 | 워드마크 | 라벨 |

칩 아래에 한 줄로:

## 7. `/color` — 컬러

## 8. `/typography` — 타이포그래피

### 8.1 목적

참가업체가 KIMES를 자료에 표기할 때 두 가지 자산(워드마크 vs 텍스트
표기)을 어떻게 다르게 쓰는지, 자유는 어디까지인지 알린다. 본문
시스템·H1~H6 같은 BI 가이드 자체의 내부 정보는 다루지 않는다.

> 글로벌 B2B 전시·기업 트렌드(KT, CES, MWC, IFA, SXSW)에서는 고정
> 워드마크는 공식·증명 자리에만 쓰고, 마케팅·콘텐츠에서는 브랜드 이름을
> 텍스트로 자유 조판하는 것이 표준입니다. KIMES도 이 방향을 따릅니다.

### 8.2 섹션 구성

#### 8.2.1 두 가지 자산 — 워드마크 vs 텍스트 표기

큰 비교 도식 1장 (좌·우 2단).

| 항목 | KIMES 워드마크 | KIMES 텍스트 표기 |
|---|---|---|
| 정의 | 공식 SVG 자산 | "KIMES" 4글자 텍스트 |
| 사용 자리 | 보증·증명 자리 | 마케팅·콘텐츠 자리 |
| 변형 | 사용 불가 (§6.2.6) | 자유 조판 (§8.2.3) |
| 폰트·색 | 고정 | 회사 디자인 시스템에 맞춰 자유 |

**워드마크를 사용하는 자리** (보증·증명):
- 보도자료 헤더 락업
- 부스 인증 마크 (KIMES 사무국 발급 자리)
- 명함의 공식 참가 표시
- 공문·계약서 헤더

**텍스트 자유 조판을 사용하는 자리** (마케팅·콘텐츠):
- 부스 그래픽
- 광고 카피
- SNS 콘텐츠
- 사내 발표 자료
- 명함 옆면 표기
- 영상 자막 등

> 두 자산은 다르게 사용합니다. 보증이 필요한 자리에는 워드마크 SVG,
> 그 외 마케팅·콘텐츠는 텍스트 자유 조판.

#### 8.2.2 텍스트 조판 — 자유롭게, 권장은 이렇게

> KIMES 텍스트 표기에는 강제가 없습니다. 회사 디자인 시스템에 맞춰
> 자유롭게 조판하셔도 됩니다.

다만 일관성을 위해 다음을 권장합니다.

- **Montserrat 800** (라틴 문자) — 권장
- **Pretendard 800** (한글) — 권장
- 색은 **KIMES Red** 또는 **KIMES Black** 단색 — 권장

권장 조판 샘플 (실제 CSS 렌더, 이미지 아님):

```
[Montserrat 800, 36px]   KIMES 2027
[Pretendard 700, 18px]   참가 · EXHIBITOR
```

회차·연도는 정확히 표기합니다 (KIMES 2027, 제42회).

## 9. `/co-branding` — KIMES + 우리 회사 로고 ★ 핵심 페이지

### 9.1 목적

**이 페이지가 가이드의 진짜 핵심**이다. 참가업체가 가장 자주 헷갈리고, 가장 많이 잘못 쓰고, 사무국이 가장 많이 정정 요청을 보내는 영역이 바로 회사 로고와 KIMES 로고를 함께 쓰는 케이스다.

### 9.2 사용자 시나리오

- "부스 헤더에 우리 로고랑 KIMES 로고 같이 넣어야 하는데 비율이 어떻게 되지?"
- "이메일 서명에 KIMES 참가 배너 넣고 싶은데 양식 있나?"
- "초청장에 두 로고 같이 넣을 때 위아래 어디가 KIMES?"

### 9.3 섹션 구성

#### 9.3.1 기본 원칙 (4줄, 큰 글씨로)

1. **KIMES 로고와 회사 로고는 시각적 무게가 비슷하게 보이도록 조정합니다.** 픽셀 비율보다 눈으로 확인하세요.
2. **두 로고 사이에는 K 높이의 1.5배 이상의 간격**을 둡니다. 1px 얇은 세로선(divider)으로 구분하는 것을 권장합니다.
3. **두 로고는 시각적 중심선으로 정렬**합니다. 베이스라인 정렬은 글자 형태가 다르면 어색합니다.
4. **KIMES 로고는 워드마크 그대로 사용**합니다. 회사 로고 색에 맞춰 변형 금지.

#### 9.3.2 측정 단위 X

다이어그램. "X = KIMES 워드마크 K의 cap-height" 표시.

#### 9.3.3 사용 케이스 6종 (이게 진짜 알맹이)

각 케이스: 큰 미리보기 이미지 + 사용 상황 설명 + 권장 비율·간격 + ✓ OK / ✗ 흔한 실수 / `[양식 다운로드]`.

각 케이스 상단에 **자산 선택 안내** 두 옵션을 함께 표시합니다.
참가업체가 자기 디자인 톤에 맞춰 둘 중 하나를 고를 수 있도록.

| 케이스 | 권장 (보증 성격이 있을 때) | 자유 (디자인 톤이 강할 때) |
|---|---|---|

| 보도자료 헤더 | 워드마크 SVG (헤더 락업) | — |
| 보도자료 본문 안 KIMES 표기 | — | 자유 조판 |
| 이메일 서명 | SVG 또는 텍스트 — 둘 다 OK | 둘 다 OK |

- ✓ 특별관 안에서의 부스·자료에서는 특별관 단독 표기 OK

#### 15.2.2 질문 30개 (아코디언)

각 질문 클릭 시 답변 펼침.

**로고**
1. KIMES 로고 색을 우리 회사 컬러에 맞춰 바꿔도 되나요?
   → 안 됩니다. KIMES 워드마크는 항상 KIMES Red, 흰색, 검정 중 하나로만 사용합니다.
2. KIMES 로고에 그림자나 글로우 효과를 넣어도 되나요?
   → 안 됩니다. 모든 효과(그림자·외곽선·입체·블러)는 금지입니다.
3. KIMES 로고 안의 검은 쐐기를 빼도 되나요?
   → 안 됩니다. 워드마크의 일부이며 임의 수정 시 사용 중단 요청 대상입니다.
4. SVG 파일 외에 PNG로만 받을 수 있나요?
   → 모든 버전이 SVG·PNG·AI·EPS로 제공됩니다. `/downloads`에서 받으세요.
5. 로고를 더 큰 해상도로 받을 수 있나요?
   → SVG와 AI는 벡터이므로 무한 확대 가능합니다. PNG는 최대 4000px 폭으로 제공됩니다.

**컬러**
6. KIMES Red의 정확한 Pantone 번호는?
   → Pantone 185 C (코팅) / 185 U (비코팅).
7. 인쇄소가 CMYK로 0/100/100/0을 찍었는데 색이 살짝 어둡습니다.
   → 종이 종류(코팅·비코팅)와 인쇄기에 따라 차이가 있습니다. Pantone 185 C 별색 매칭을 권장합니다.

**표기**
8. "KIMES"를 "킴스"로 한글 표기해도 되나요?
   → 안 됩니다. 영문 KIMES 그대로 사용합니다.
9. "제42회"를 "42nd"로 영문 표기할 때 형식은?
   → "KIMES 2027 — 42nd edition" 형식.
10. 보도자료에 KIMES 풀네임을 매번 써야 하나요?
    → 처음 등장할 때만 풀네임. 두 번째부터는 "KIMES"만 사용.

**함께 쓰기**
11. 부스 헤더에 우리 회사 로고와 KIMES 로고 비율은?
    → 시각적 무게가 비슷해 보이도록. 일반적으로 회사 : KIMES = 1 : 0.7. `/co-branding` 참고.
12. 이메일 서명에 KIMES 로고를 넣고 싶은데 양식이 있나요?
    → `/co-branding` 또는 `/applications`에서 다운로드.
13. 명함에 KIMES 표기를 넣어도 되나요?
    → 행사 기간 한정으로 가능. 행사 종료 후 재인쇄 시 제거 권장.
14. SNS 게시물에 KIMES 로고 들어간 걸 올릴 때 별도 승인 필요한가요?
    → 일반적인 게시물은 별도 승인 불필요. 다만 사무국 사전 검토를 권장합니다(의무 아님).

**권한·승인**
15. KIMES 로고가 들어간 굿즈를 만들어도 되나요?
    → 사전 승인 필요. kimes@kimes.kr.
16. 회사 광고에 KIMES 로고를 넣고 싶습니다.
    → 사전 승인 필요. 매체·기간·시안 첨부.
17. "KIMES 공식 파트너"라고 표기해도 되나요?
    → 별도 라이선스 계약 필요. 무단 사용 시 법적 조치 대상.
18. 사장님이 KIMES 사진을 회사 인터뷰에 쓰고 싶어 합니다.
    → 사진 종류·노출 매체에 따라 다름. kimes@kimes.kr 문의.
19. 영상 인트로에 KIMES 로고 1초 노출은 승인 필요한가요?
    → 노출 시간과 무관하게 사전 승인 필요.
20. 행사 끝나고도 회사 홈페이지에 KIMES 참가 표기를 유지해도 되나요?
    → 차회 참가가 확정되지 않았다면 "Past Exhibitor" 표기로 변경 또는 제거.

**특별관**

**다운로드·기술**

    → 한국어 파일명 인코딩 문제일 수 있습니다. 다른 압축 해제 프로그램(반디집·7-Zip)을 써보세요.

    → SVG·EPS로 받아 사용하거나 무료 도구(Inkscape)로 열 수 있습니다.

### 21.1 반응형 원칙

- 모바일 퍼스트
- 모든 표는 가로 스크롤 가능 + 모바일에서 카드 형식 대안 제공

- 사이드 TOC는 모바일에서 상단 셀렉트로 변환

### 21.2 접근성 (WCAG 2.1 AA 준수)

- 모든 인터랙티브 요소에 포커스 링 (KIMES Red 2px outline + 2px offset)
- 키보드 네비 가능
- 이미지 alt 텍스트 필수
- 색만으로 의미 전달 금지 (✓/✗ 아이콘 + 텍스트 라벨 병기)
- prefers-reduced-motion 존중 (모션 자동 재생 없음)
- 최소 히트 영역 44×44px

### 21.3 SEO

- 페이지마다 고유 `<title>`, `<meta description>`
- og:image: 페이지별 프리뷰 (1200×630)
- 구조화 데이터 (JSON-LD)
  - 행사 정보: schema.org/Event
  - 조직: schema.org/Organization
  - FAQ 페이지: schema.org/FAQPage
- 사이트맵 자동 생성
- robots.txt: `/contact`, `/downloads` 크롤링 허용 / `/changelog` 허용

### 21.4 다국어

- 1차: 한국어 우선
- 2차 (후속 작업): 영어 풀버전. 라우트 `/en/...` 분리. URL 상단 언어 토글.
- 영어 버전이 준비되기 전까지는 핵심 페이지(`/overview`, `/notation`, `/permissions`)에 영문 보일러플레이트만 인라인으로 제공.

### 21.5 인쇄 (Print CSS)

- 가이드 전체를 PDF로 인쇄 가능하도록 print.css 별도 작성
- 내비·CTA·다운로드 버튼은 인쇄 시 숨김
- 페이지 번호·갱신일 푸터 자동 삽입

## 22. 콘텐츠 톤 & 보이스 (작성 규칙)

가이드는 **권리 관계 문서**이지 광고 카피가 아닙니다. 동시에 참가업체가
자기 디자인을 펼치는 자유는 충분히 보장합니다. 톤은 차분하고 단정적이되,
강제·금지 표현은 진짜 강제가 필요한 자리에만 사용합니다.

### 22.1 어조

- **차분하고 단정적인 "~합니다" 체** — 친근체("~해요"), 명령형("~해라")은 사용하지 않습니다
- 1인칭("저희"), 2인칭("귀사") 가능. 기본은 무인칭
- 한 문단 = 한 의미. 길어지면 잘라 씁니다

### 22.2 강제 표현은 4개 케이스에만

"반드시·절대·금지·사용 불가" 같은 강한 표현은 다음 4개 케이스에서만
사용합니다. 모두 객관적으로 위반 여부를 판단할 수 있는 항목입니다.
그 외 모든 권고는 "권장합니다 / 권장하지 않습니다 / 되도록"으로
부드럽게 표현합니다.

| # | 절대 금지 케이스 | 위치 |
|---|---|---|
| 1 | KIMES 표기 자체 변형 (Kimes / 킴스 / 케이아이엠이에스 등) | §10 표기 규칙 |
| 2 | 워드마크 SVG 변형 (색·비례·쐐기 제거 등 11종) | §6.2.6 |
| 3 | 미승인 라이선스 표현 ("KIMES 공식 파트너" 등) | §9.3.4 |
| 4 | 회차·연도 오기 (2026 vs 2027 혼동, 제42회 오기) | §10 |

이 4개 외에는 "권장 / 권장하지 않음" 톤. 텍스트 자유 조판, 디자인 톤
선택, 다양한 사용 케이스 등은 모두 자유 영역.

**주관 판단 영역은 가드레일 박스로 분리**

사행성·풍자성 톤 KIMES 표기처럼 주관 판단이 필요한 케이스는 §8.2.5
가드레일 박스에서 부드러운 톤으로 안내합니다 ("사무국이 정정을
요청드릴 수 있습니다"). 절대 금지 카테고리에는 들어가지 않습니다.

### 22.3 어휘 권장 — 검토 플래그 (강제 아님)

다음 표현이 본문에 등장하면 자동 검수가 플래그를 띄웁니다. 의도된
사용이면 그대로 두고, 무의식적 사용이면 권장 대체로 바꿉니다.

| 검토 플래그 | 권장 대체 | 비고 |
|---|---|---|
| ~을/를 가집니다 | ~이 있습니다 / 있다 | 일본어식 표현 |
| 프라이머리 / 세컨더리 | 메인 / 보조 | 외래어 음차 |
| 베리언트 | 변형 / 버전 | 외래어 음차 |
| 컨텍스트 | 상황 / 쓰임 | 외래어 음차 |
| ~ 처리입니다 | ~ 합니다 / 적용합니다 | 사무적 명사화 |
| ~ 표면입니다 | ~ 화면입니다 / ~ 영역입니다 | 디자인 시스템 용어 |
| 본 시스템 / 본 가이드 | 이 가이드 | 격식 과잉 |
| 디자인 시스템 (자칭) | BI 가이드 | 자칭 회피 |
| 이슈 | 문제 | 외래어 음차 |
| 어카이브 | 보관 | 외래어 음차 |
| 컬러풀 / 클린한 / 모던한 | 다채로운 / 깔끔한 / 현대적인 | 마케팅 클리셰 |

### 22.4 자산 혼동 방지 (검수 플래그)

- "다른 폰트로 워드마크 작성" → §8.2.5 가드레일 위반 권장하지 않음

### 22.6 한국어 톤 정제 — 강한 단어 정리

한국어 가이드에서 강한 명령조 표현은 부자연스럽고 위협적으로 읽힙니다
(영어 "Don't" 직역식 사고). 한국 대표 디자인 가이드(카카오·네이버·삼성·
토스)는 모두 부드러운 권장 톤입니다. KIMES 가이드 본문에서도 다음 단어를
사용하지 않습니다.

**가이드 본문에서 사용하지 않을 단어**

| 사용하지 않음 | 대신 |
|---|---|
| 절대 | (삭제) |
| 반드시 (대부분의 경우) | 일반적으로 / 기본적으로 / (삭제) |
| 금지 | 권장하지 않음 / 사무국 협의 |
| 위반 | (가이드 본문에서 삭제 — 사무국 내부 용어로만) |
| 하지 말 것 | 피해주세요 / 자제해주세요 |
| 불가 / 사용 불가 | 사용 제한 |

**대신 사용**

- 권장합니다 / 권장하지 않습니다
- 피해주세요 / 자제해주세요
- 사무국과 협의해주세요
- "이런 사용은 자제해주세요" / "이런 톤이 잘 맞습니다"
- 라벨이 필요하면 영문 한 단어 ("Don't", "Restricted" 등)

**예외 — §22.2 4개 절대 금지 케이스**

§22.2의 객관 판단 가능한 4개 케이스(KIMES 표기 변형 / 워드마크 SVG 변형
/ 미승인 라이선스 표현 / 회차·연도 오기)에서만 강한 표현 허용. 그 외에는
모두 부드러운 톤.

**참고할 한국 기업 가이드 톤**

- 카카오 디자인: "이렇게 사용해주세요" / "이렇게는 사용하지 않습니다"
- 네이버: "권장 사용" / "피해주세요"
- 삼성: "Do" / "Don't" 영문 라벨
- 토스: "이런 사용은 자제해주세요"

## 25. 마치며

이 가이드의 성공 기준은 단 하나다.

**"참가업체가 사무국에 BI 관련 문의 메일을 보내는 빈도가 줄었는가."**

가이드가 충분히 명확하고 실용적이면 문의가 줄어든다. 줄지 않았다면 가이드에 빠진 케이스가 있는 것이다. 그 케이스들을 모아서 다음 버전에 추가한다.

가이드는 한 번 만들고 끝나는 문서가 아니다. 매년 행사 끝나고 사무국이 받은 문의·정정 요청을 모아 다음 버전에 반영한다. 이게 BI 가이드의 라이프사이클이다.
