## 0. 이 문서의 사용법 (Claude Code에게)

이 명세서는 위에서 아래로 순서대로 작업하면 된다. 각 섹션은 다음 구조를 따른다.

- **목적**: 왜 이 페이지/섹션이 존재하는가
- **사용자 시나리오**: 참가업체가 어떤 상황에서 이걸 보는가
- **콘텐츠 명세**: 페이지에 들어갈 모든 텍스트·표·UI 요소
- **인터랙션**: 버튼·복사·다운로드·필터 등의 동작
- **금지사항**: 절대 들어가면 안 되는 것

작업 순서:
1. 먼저 §1, §2 (삭제 대상, 새 IA)를 읽고 현재 코드베이스에서 들어낼 것부터 정리
2. 그 다음 §3~§13의 페이지별 명세에 따라 새 페이지를 빌드
3. 마지막으로 §14 (디자인 시스템 토큰), §15 (반응형·접근성), §16 (배포 체크리스트) 적용

## 1. 작업의 본질 — 이걸 먼저 이해하지 않으면 다 망친다

### 1.1 가이드의 진짜 목적

이 가이드는 **"KIMES 브랜드를 외부가 함부로 쓰지 못하게 통제하면서, 참가업체에게는 충분히 활용할 수 있도록 권한을 명확히 정의하는 법적·실무적 매뉴얼"** 이다.

디자인 시스템이 아니다. 디자인 가이드도 절반의 정의일 뿐이다.

전시 BI 가이드의 본질은 **권리 관계 문서**다. 참가업체가 자기 마케팅에 KIMES 자산을 쓸 때, 어디까지 허용되고 어디부터는 승인이 필요하며, 어떤 행위는 절대 금지인지 — 이걸 흐릿하게 두면 1,400개 참가업체가 1,400개 다른 해석을 한다.

### 1.2 사용자

**유일한 사용자**: KIMES 2027 참가업체와 그 대행사·협력사.

세부적으로는:
- 참가업체 마케팅 담당자 (보도자료·SNS·홈페이지)
- 참가업체 디자인 팀 (부스·인쇄물·굿즈)
- 외주 부스 디자인 회사
- PR 에이전시
- 영상·SNS 콘텐츠 제작사
- (특별관 참가업체) 특별관 BI까지 추가로 따라야 함

**비대상**: 일반 관람객, 언론사, KIMES 사무국 내부, 디자인 시스템 운영팀.

### 1.3 4대 원칙 (모든 작업의 판단 기준)

1. **번역투 박멸** — 영어를 직역한 표현 전부 자연스러운 한국어로. ("프라이머리/세컨더리/베리언트/컨텍스트/처리/표면" 같은 음차·직역 금지)
2. **KIMES 단일 메인** — KIMES만 메인 브랜드. MedicomteK·Beauty&Derma·INSPIRE는 "특별관(Special Zone)"으로 정의. 4-브랜드 평행 구조 폐기.
3. **참가업체 외 자료 전부 삭제** — 사이니지·부스 마커·등록 데스크·모션 키트·BI 감사·내부 컴포넌트 시스템 전부 제거
4. **권리 관계 명시** — "할 수 있다 / 승인 필요 / 절대 금지" 3단계가 모든 행위에 명확히 라벨링되어야 함

### 2.2 표현·용어 단위 삭제 (전역 검색·치환)

| 삭제 대상 | 처리 |
|---|---|
| "슬래시컷" / "Slash-cut" / "슬래시컷 패밀리" | 전부 삭제. i 쐐기는 §3 로고 페이지에서 "워드마크의 디테일"로만 한 번 언급 |
| "프라이머리 / 세컨더리" | "메인 / 보조"로 치환 |
| "베리언트" | "변형" 또는 "버전" |
| "컨텍스트" | "상황" 또는 "쓰임새" |
| "토큰" (사용자 노출) | 내부 코드에서만 사용. 본문에서는 "값" 또는 항목명 |
| "Independent identity" | 전부 삭제 |
| "Pattern A/B/C" | 실제 사용 케이스명으로 치환 (예: "부스 헤더 표기", "보도자료 표기") |
| "~을/를 가집니다" (have 직역) | "~이 있습니다" 또는 능동형 |
| "~ 처리입니다 / ~ 표면입니다" | 자연스러운 동사형으로 |
| "본 시스템", "본 가이드" | "이 가이드" |
| "디자인 시스템" (이 사이트 자칭) | "BI 가이드" |

## 5. `/overview` — KIMES 한눈에 보기

### 5.1 목적

참가업체가 보도자료·홈페이지·SNS·이메일에 그대로 **복사해서 쓸 수 있는 공식 문구**를 한 페이지에 다 모은다. 이 페이지가 진짜 "참가업체용 가이드"의 시작점이다.

### 5.2 사용자 시나리오

- "보도자료 쓰는데 KIMES 정식 영문명이 뭐였지?"
- "회사 홈페이지에 KIMES 참가 알림 띄우는데 한 줄 소개가 필요해"
- "사장님 영문 인사말에 KIMES 참가 사실을 어떻게 표기하지?"

### 5.3 섹션 구성

#### 5.3.1 공식 명칭

표 형식. 각 행 우측에 `[복사]` 버튼. 클립보드 복사 + 토스트 ("복사되었습니다").

|---|---|---|
| 정식 | 제42회 국제 의료기기·병원설비 전시회 KIMES 2027 | KIMES 2027 — Korea International Medical & Hospital Equipment Show |
| 줄임 | KIMES 2027 | KIMES 2027 |
| 약칭 | KIMES | KIMES |

#### 5.3.2 일정·장소

| 구분 | 한국어 | 영문 |
|---|---|---|
| 기간 | 2027년 3월 18일(목) ~ 21일(일) | March 18–21, 2027 |
| 장소 | 코엑스 (서울 강남구 영동대로 513) | COEX, Seoul |

#### 5.3.3 한 줄 소개 (보일러플레이트)

길이별로 4개 버전. 보도자료 끝에 들어가는 표준 회사 소개문(boilerplate)처럼 쓸 수 있게.

**한국어**
- **40자 이내** — SNS·이메일 제목용
  > 한국 최대 의료기기 전시회 KIMES 2027, 3월 18~21일 코엑스
- **100자 이내** — 보도자료 리드문용
  > KIMES 2027은 1,400여 개 기업과 8만여 명의 전문 관람객이 참여하는 한국 최대 규모의 의료기기 전시회입니다.
- **200자 이내** — 회사 소개 페이지·블로그용
  > KIMES는 한국의료기기산업협회가 1980년부터 매년 개최해 온 대한민국 대표 의료기기·병원설비 전시회입니다. 2027년에는 제42회를 맞아 60여 개국 1,400여 개 기업이 참가하며, 영상진단·수술·치료·재활·뷰티·디지털 헬스 등 14개 카테고리를 아우릅니다.
- **400자 이내** — 보도자료 회사 소개 칸용

**영문**
- 동일하게 4개 길이.

각 박스 우측 상단에 `[복사]` 버튼, 좌측 하단에 글자 수 표시.

#### 5.3.4 핵심 숫자

## 6. `/logo` — 로고

### 6.1 목적

KIMES 워드마크를 정확하게 다운로드하고, 어떤 상황에서 어떤 버전을 써야 하는지 알려준다. 변형·재타이핑·임의 색상 변경을 명확히 막는다.

### 6.2 섹션 구성

#### 6.2.1 KIMES 워드마크

- 큰 이미지 1장 (워드마크 단독, 흰 배경)
- 짧은 설명 (3~4줄):
  > KIMES 워드마크는 1980년 첫 행사 이래 단단하고 직선적인 산세리프로 한국 의료기기 산업의 무게감을 표현해왔습니다. 강렬한 빨강과 두꺼운 글자, 그리고 i 안에 자리 잡은 검은 쐐기 — 이 세 요소가 KIMES만의 시각적 정체성입니다.

  ※ "슬래시컷 패밀리" 같은 표현은 절대 쓰지 않는다. i 쐐기는 "워드마크의 디테일"로만 언급.

#### 6.2.2 로고 버전 (그리드 카드)

5개 카드. 각 카드: 미리보기 + 라벨 + 다운로드 버튼군.

| 버전 | 용도 | 파일 |
|---|---|---|
| 기본 (레드) | 흰 배경·밝은 사진 위 | SVG · PNG · AI · EPS |
| 화이트 | 빨간색·검정·어두운 배경 위 | SVG · PNG · AI · EPS |
| 블랙 | 단색 인쇄·팩스·복사 | SVG · PNG · AI · EPS |
| 그레이 | 본문 안 작은 표기·각주 | SVG · PNG · AI · EPS |
| 연도 일체형 (KIMES 2027) | 행사 직전 발표·초청장 | SVG · PNG · AI · EPS |

각 카드 하단 미세글씨: `해상도: 벡터 / 컬러모드: SVG = sRGB · AI = CMYK + Pantone`

#### 6.2.3 클리어 스페이스

큰 다이어그램 1개.

- 워드마크 주변에 점선 박스 + "1X = K의 높이"
- 글: "워드마크 사방으로 K 높이만큼의 빈 영역을 반드시 확보합니다. 이 영역 안에는 다른 로고·문구·이미지가 들어갈 수 없습니다."

#### 6.2.4 최소 크기

- 디지털: 가로 80px 이상 (이메일 서명·웹 푸터 등에서)
- 인쇄: 가로 24mm 이상 (명함·리플렛 등)
- 사이니지·대형 인쇄: 별도 제한 없음 (단, 명도·해상도 확인 권장)

#### 6.2.5 배경별 사용 매트릭스

표 형식. 각 행 우측에 권장 버전 + ✓ / ✗ 표시.

| 배경 | 권장 버전 | 비고 |
|---|---|---|
| 흰색 또는 밝은 단색 (밝기 ≥ 70%) | 기본 (레드) | 가장 일반적 |
| 중간 명도 단색 (40~60%) | 가급적 회피, 사용 시 사전 승인 | 가독성 저하 |
| 어두운 단색 (≤ 30%) | 화이트 | 빨간 로고 ✗ |
| 빨간색 배경 | 화이트 | 기본 레드 절대 금지 |
| 회사 브랜드 컬러 위 | 화이트 (밝은 회사색이면 블랙) | — |
| 단색 인쇄·팩스 | 블랙 | — |
| 밝고 단순한 사진 | 기본 (레드) | 클리어 스페이스 확보 필수 |
| 어두운 사진 | 화이트 | — |
| 복잡하고 디테일 많은 사진 | 단색 플레이트 위에 올린 후 사용 | 직접 얹지 않음 |

#### 6.2.6 절대 하지 말 것 (Don'ts) — 12종

3×4 그리드. 각 셀: ✗ 마크 + 잘못된 예시 + 한 줄 설명.

1. 색 바꾸기 (회사 컬러로 통일)
2. i 안의 검은 쐐기 제거
3. 가로·세로 늘리기
4. 회전·기울이기
5. 그림자·외곽선·글로우
6. 그라디언트 채움
7. 다른 폰트로 다시 타이핑 (CSS 텍스트 포함)
8. 원·박스·배지 안에 가두기
9. 입체·3D 효과
10. 워드마크 일부 잘라내기
11. 회사 로고와 합쳐 새 로고 만들기
12. 패턴·텍스처 채움

각 셀 하단 작은 글씨: `위반 시 사용 중단 요청 + 재발 시 참가 자격 검토 대상`

#### 6.2.7 로고 사용 신청

페이지 끝에 박스.

> 가이드에 명시된 일반적인 사용은 별도 신청이 필요 없습니다.

> - 가이드에 없는 새로운 사용 케이스

## 8. `/typography` — 타이포그래피 (참가업체용 최소)

### 8.1 목적

참가업체가 KIMES 표기를 만들 때 어떤 폰트로 어떻게 조판해야 하는지만 알려준다. 본문 시스템·H1~H6 위계 같은 디자인 시스템 내부 정보는 다루지 않는다.

### 8.2 섹션 구성

#### 8.2.1 사용 서체 2종

- **Montserrat** — 라틴 문자, 숫자, KIMES 워드마크 주변 영문 보조 텍스트
  - 오픈소스. Google Fonts에서 다운로드.
  - 굵기 추천: KIMES 표기 = 800 (ExtraBold) / 본문 = 400 (Regular)
- **Pretendard** — 한글 본문, 한·영 혼용 텍스트
  - 오픈소스. GitHub에서 다운로드.
  - 굵기 추천: 헤드라인 = 800 / 본문 = 400

각 카드에 `[다운로드]` 외부 링크.

#### 8.2.2 한·영 혼용 4가지 규칙

각 항목: ✓ OK 예시 / ✗ 잘못된 예시 + 1줄 설명.

1. **숫자는 항상 Montserrat**
   ✓ `전시기간 March 18—21, 2027`
   ✗ `전시기간 March 18—21, 2027` (Pretendard로 숫자)

2. **약어·브랜드명은 항상 Montserrat**
   ✓ `AI 기반 진단 / KIMES 사무국`
   ✗ `AI 기반 진단` (한글 폰트로 영문)

3. **같은 줄에서는 굵기와 크기를 맞춘다**
   ✓ `참가 기업  EXHIBITORS` (둘 다 Bold)
   ✗ `참가 기업  EXHIBITORS` (한글 Regular + 영문 Bold)

4. **한글 본문은 양쪽 정렬 금지**
   - 한글 본문은 좌측 정렬이 기본
   - 가운데 정렬은 짧은 헤드라인에만
   - `word-break: keep-all` 사용 (어절 단위 줄바꿈)

#### 8.2.3 KIMES 표기 조판 표준

이 부분이 핵심. 참가업체 자료에 "KIMES 2027 참가" 표기를 만들 때 따라야 할 것.

```
[Montserrat 800, 18pt]   KIMES 2027
[Pretendard 700, 11pt]   참가 · EXHIBITOR
```

샘플 이미지로 보여주기.

권장 사항:
- "KIMES 2027"은 항상 Montserrat 800
- 한글 보조 라벨("참가", "공식 파트너" 등)은 Pretendard 700
- 두 줄 사이 간격은 KIMES 글자 높이의 0.4배
- 색은 KIMES Red 또는 KIMES Black 중 하나만

#### 8.2.4 제3의 서체 사용 금지

> KIMES 표기에는 Montserrat과 Pretendard만 사용합니다. 회사 자체 서체나 제3의 서체로 "KIMES"를 다시 타이핑하지 마세요. 워드마크는 반드시 제공된 SVG/AI 파일을 그대로 사용합니다.

## 9. `/co-branding` — KIMES + 우리 회사 로고 ★ 핵심 페이지

### 9.1 목적

**이 페이지가 가이드의 진짜 핵심**이다. 참가업체가 가장 자주 헷갈리고, 가장 많이 잘못 쓰고, 사무국이 가장 많이 정정 요청을 보내는 영역이 바로 회사 로고와 KIMES 로고를 함께 쓰는 케이스다.

### 9.2 사용자 시나리오

- "부스 헤더에 우리 로고랑 KIMES 로고 같이 넣어야 하는데 비율이 어떻게 되지?"
- "이메일 서명에 KIMES 참가 배너 넣고 싶은데 양식 있나?"
- "초청장에 두 로고 같이 넣을 때 위아래 어디가 KIMES?"

### 9.3 섹션 구성

#### 9.3.1 기본 원칙 (4줄, 큰 글씨로)

1. **KIMES 로고와 회사 로고는 시각적 무게가 비슷하게 보이도록 조정합니다.** 픽셀 비율보다 눈으로 확인하세요.
2. **두 로고 사이에는 K 높이의 1.5배 이상의 간격**을 둡니다. 1px 얇은 세로선(divider)으로 구분하는 것을 권장합니다.
3. **두 로고는 시각적 중심선으로 정렬**합니다. 베이스라인 정렬은 글자 형태가 다르면 어색합니다.
4. **KIMES 로고는 워드마크 그대로 사용**합니다. 회사 로고 색에 맞춰 변형 금지.

#### 9.3.2 측정 단위 X

다이어그램. "X = KIMES 워드마크 K의 cap-height" 표시.

#### 9.3.3 사용 케이스 6종 (이게 진짜 알맹이)

각 케이스: 큰 미리보기 이미지 + 사용 상황 설명 + 권장 비율·간격 + ✓ OK / ✗ 흔한 실수 / `[양식 다운로드]`.

##### 케이스 1. 부스 헤더 / 사이니지

```
[회사 로고]   │   [KIMES 2027 참가]
                  EXHIBITOR
```

- **상황**: 부스 정면 상단, 부스 입구 위 광폭 사이니지
- **권장 비율**: 회사 로고 : KIMES = 1 : 0.7 (시각적 무게 균형)
- **간격**: 1.5X
- **구분선**: 1px 세로선 (KIMES Black 또는 #E1E5EA)
- **위치**: 회사 로고가 좌측, KIMES가 우측 (한국에서 좌→우로 읽는 흐름)
- ✓ OK: 회사 로고가 메인, KIMES는 참가 사실 표시
- ✗ 흔한 실수: KIMES 로고를 회사 컬러로 변경, KIMES를 회사 로고보다 크게
- `[부스 헤더 양식 .ai 다운로드]`

##### 케이스 2. 보도자료 헤더

```
[회사 로고]                                  보도자료 / PRESS
─────────────────────────────────────────────────────────
                                              KIMES 2027 참가 발표
                                              March 18–21 · COEX
```

- **상황**: 보도자료 PDF 또는 워드 파일 상단
- KIMES 로고는 헤더에 넣지 않음. 본문 안에 텍스트 표기.
- 본문 첫 문단에 표준 표기 삽입:
  > 당사는 2027년 3월 18일부터 21일까지 코엑스에서 열리는 KIMES 2027에 참가합니다. (Booth A-101)
- 보도자료 끝 보일러플레이트에 KIMES 한 줄 소개 (`/overview`에서 복사)
- `[보도자료 템플릿 .docx]`

##### 케이스 3. 이메일 서명 배너

```
─────────────────────────────────────
[회사 로고]   홍길동 / 마케팅팀장

```

- **권장 사이즈**: 600×120px (고정 폭)
- KIMES 표기 박스: 흰 배경에 KIMES Red 워드마크 + 일정·부스
- `[이메일 서명 양식 .png 4종]` (Outlook · Gmail · Apple Mail · 모바일 호환)
- 1줄 안내: HTML 서명을 만들 수 있다면 `<img>`로, 아니면 PNG로 첨부

#### 14.2.2 행위 시점

행사 기간 표기 vs 상시 표기 구분.

> "KIMES 2027 참가" 표기는 행사 종료 시점(2027년 3월 21일)까지 유효합니다.
> 행사 종료 후에는 회사 자료에서 KIMES 로고를 제거하거나 "지난 KIMES 2027 참가 (Past Exhibitor)" 표기로 변경합니다.
> 다음 회차 참가가 확정된 경우 새 회차 표기로 갱신합니다.

#### 14.2.3 신청 절차

3단계 카드.

```
1. 신청서 작성    →    2. 검토 (영업일 3~5일)    →    3. 승인 회신
```

신청서 양식:
- 신청 회사
- 담당자 / 연락처
- 사용 목적 (굿즈·영상·광고 등 선택)
- 사용 매체·노출 채널
- 사용 기간
- 시안 (PDF·이미지 첨부)
- 부스 번호 (해당 시)

`[승인 신청서 다운로드 .docx]` `[온라인 신청 →]`

#### 14.2.4 위반 시 조치

박스 강조.

> 가이드에 명시된 권한 범위를 초과하거나 미승인 표기를 사용한 경우, 다음 절차를 따릅니다:

> 3. **3차** — 차회 KIMES 참가 자격 검토 및 법적 조치

> 의도적 무단 사용·위조의 경우 1차 절차를 생략할 수 있습니다.

## 15. `/faq` — 자주 묻는 질문

### 15.1 목적

사무국이 1년 동안 받는 같은 질문 30개를 미리 답해두는 페이지. 검색·카테고리 필터 제공.

### 15.2 섹션 구성

#### 15.2.1 검색 + 카테고리 필터

- 상단 검색창 (실시간 필터)
- 카테고리 칩: 전체 / 로고 / 컬러 / 표기 / 함께쓰기 / 권한·승인 / 특별관 / 다운로드 / 행사 일반

#### 15.2.2 질문 30개 (아코디언)

각 질문 클릭 시 답변 펼침.

**로고**
1. KIMES 로고 색을 우리 회사 컬러에 맞춰 바꿔도 되나요?
   → 안 됩니다. KIMES 워드마크는 항상 KIMES Red, 흰색, 검정 중 하나로만 사용합니다.
2. KIMES 로고에 그림자나 글로우 효과를 넣어도 되나요?
   → 안 됩니다. 모든 효과(그림자·외곽선·입체·블러)는 금지입니다.
3. KIMES 로고 안의 검은 쐐기를 빼도 되나요?
   → 안 됩니다. 워드마크의 일부이며 임의 수정 시 사용 중단 요청 대상입니다.
4. SVG 파일 외에 PNG로만 받을 수 있나요?
   → 모든 버전이 SVG·PNG·AI·EPS로 제공됩니다. `/downloads`에서 받으세요.
5. 로고를 더 큰 해상도로 받을 수 있나요?
   → SVG와 AI는 벡터이므로 무한 확대 가능합니다. PNG는 최대 4000px 폭으로 제공됩니다.

**컬러**
6. KIMES Red의 정확한 Pantone 번호는?
   → Pantone 185 C (코팅) / 185 U (비코팅).
7. 인쇄소가 CMYK로 0/100/100/0을 찍었는데 색이 살짝 어둡습니다.
   → 종이 종류(코팅·비코팅)와 인쇄기에 따라 차이가 있습니다. Pantone 185 C 별색 매칭을 권장합니다.

**표기**
8. "KIMES"를 "킴스"로 한글 표기해도 되나요?
   → 안 됩니다. 영문 KIMES 그대로 사용합니다.
9. "제42회"를 "42nd"로 영문 표기할 때 형식은?
   → "KIMES 2027 — 42nd edition" 형식.
10. 보도자료에 KIMES 풀네임을 매번 써야 하나요?
    → 처음 등장할 때만 풀네임. 두 번째부터는 "KIMES"만 사용.

**함께 쓰기**
11. 부스 헤더에 우리 회사 로고와 KIMES 로고 비율은?
    → 시각적 무게가 비슷해 보이도록. 일반적으로 회사 : KIMES = 1 : 0.7. `/co-branding` 참고.
12. 이메일 서명에 KIMES 로고를 넣고 싶은데 양식이 있나요?
    → `/co-branding` 또는 `/applications`에서 다운로드.
13. 명함에 KIMES 표기를 넣어도 되나요?
    → 행사 기간 한정으로 가능. 행사 종료 후 재인쇄 시 제거 권장.
14. SNS 게시물에 KIMES 로고 들어간 걸 올릴 때 별도 승인 필요한가요?
    → 일반적인 게시물은 별도 승인 불필요. 다만 사무국 사전 검토를 권장합니다(의무 아님).

**권한·승인**
15. KIMES 로고가 들어간 굿즈를 만들어도 되나요?
    → 사전 승인 필요. brand@kimes.kr.
16. 회사 광고에 KIMES 로고를 넣고 싶습니다.
    → 사전 승인 필요. 매체·기간·시안 첨부.
17. "KIMES 공식 파트너"라고 표기해도 되나요?
    → 별도 라이선스 계약 필요. 무단 사용 시 법적 조치 대상.
18. 사장님이 KIMES 사진을 회사 인터뷰에 쓰고 싶어 합니다.
    → 사진 종류·노출 매체에 따라 다름. brand@kimes.kr 문의.
19. 영상 인트로에 KIMES 로고 1초 노출은 승인 필요한가요?
    → 노출 시간과 무관하게 사전 승인 필요.
20. 행사 끝나고도 회사 홈페이지에 KIMES 참가 표기를 유지해도 되나요?
    → 차회 참가가 확정되지 않았다면 "Past Exhibitor" 표기로 변경 또는 제거.

**특별관**
21. 우리는 MedicomteK 특별관에 참가합니다. KIMES 로고를 안 써도 되나요?
    → 외부 자료(보도자료·홈페이지)에는 KIMES 로고를 함께 표기합니다. 특별관 단독 표기는 특별관 안에서만.
22. 특별관 로고는 어디서 받나요?
    → `/special-zones`에서 각 특별관별로 다운로드.

**다운로드·기술**
23. 로고 파일 압축 해제가 안 됩니다.
    → 한국어 파일명 인코딩 문제일 수 있습니다. 다른 압축 해제 프로그램(반디집·7-Zip)을 써보세요.
24. AI 파일을 못 여는데 다른 형식 있나요?
    → SVG·EPS로 받아 사용하거나 무료 도구(Inkscape)로 열 수 있습니다.

**행사 일반**
25. 부스 위치는 어떻게 정해지나요?
    → 참가 신청 순서·카테고리·이전 회차 참가 이력에 따라 사무국이 배정.
26. 사전등록 마감일은?
    → 2027년 3월 17일 23:59 (행사 시작 전날).
27. 부스 설치 가능 시간은?
    → 행사 전일 09:00~21:00. 자세한 운영 매뉴얼은 참가업체 메뉴얼 참조.
28. 참가업체 출입증 발급은?
    → 사전등록 시 자동 발급. 행사 첫날 East Gate 참가업체 데스크 수령.
29. 외국인 참가업체 비자 지원이 있나요?
    → 초청장 발급 가능. visit@kimes.kr 문의.
30. 가이드에 없는 사용 케이스가 있습니다.
    → brand@kimes.kr 문의. 영업일 3~5일 안에 답변.

## 22. 콘텐츠 톤 & 보이스 (작성 규칙)

이 섹션을 어기면 가이드 전체가 무너진다. 모든 본문은 다음 규칙을 따른다.

### 22.1 어조

- **차분하고 단정적인 "~합니다" 체** — 친근체("~해요") 금지, 명령형("~해라") 금지
- 1인칭("저희") 가능, 2인칭("귀사") 가능. 기본은 무인칭.
- 한 문단 = 한 의미. 길어지면 잘라 쓴다.

### 22.2 금지 표현 (전역 검색·치환 대상)

| 금지 | 대체 |
|---|---|
| ~을/를 가집니다 | ~이 있습니다 / 있다 |
| 프라이머리 / 세컨더리 | 메인 / 보조 |
| 베리언트 | 변형 / 버전 |
| 컨텍스트 | 상황 / 쓰임 |
| ~ 처리입니다 | ~ 합니다 / 적용합니다 |
| ~ 표면입니다 | ~ 화면입니다 / ~ 영역입니다 |
| 본 시스템 / 본 가이드 | 이 가이드 |
| 디자인 시스템 (자칭) | BI 가이드 |
| 이슈 | 문제 |
| 어카이브 | 보관 |
| 픽처 / 이미지 | 사진 / 이미지 (이미지는 OK) |
| 컬러풀 | 다채로운 |
| 클린한 | 깔끔한 |
| 모던한 | 현대적인 |

### 22.3 권장 표현

- "~할 수 있습니다" / "~합니다" / "~해주세요"
- 능동형 우선, 수동형 최소
- 기술 용어는 처음 등장 시 한 번 풀어 쓰고 이후 약어
- 외래어는 음차보다 한국어 표현 우선 (단, 정식 명칭은 그대로)

### 22.4 한 줄 길이

- 본문 한 줄 권장 60자 이내
- 헤드라인은 30자 이내
- 표 셀은 셀 폭의 80% 이내

## 25. 마치며

이 가이드의 성공 기준은 단 하나다.

**"참가업체가 사무국에 BI 관련 문의 메일을 보내는 빈도가 줄었는가."**

가이드가 충분히 명확하고 실용적이면 문의가 줄어든다. 줄지 않았다면 가이드에 빠진 케이스가 있는 것이다. 그 케이스들을 모아서 다음 버전에 추가한다.

가이드는 한 번 만들고 끝나는 문서가 아니다. 매년 행사 끝나고 사무국이 받은 문의·정정 요청을 모아 다음 버전에 반영한다. 이게 BI 가이드의 라이프사이클이다.
