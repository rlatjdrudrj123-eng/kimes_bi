## 2026-05-08 — §6.2.5 4-바 시스템 + 사각지대 제거 + 칩 라벨 정정

### 정정 내용

§6.2.5 시각 검증에서 5가지 정정 + 추가 정정 1건. 명도 바 3개 → 4개로
확장(Gray 추가), White/Black 워드마크는 KT BI 가이드 시스템을 차용해
**명도 + 컬러 종합 그라디언트 바**로 변경. 컬러 시뮬레이션은 칩 5개
사이 16px 여백 + 라벨을 컬러 이름이 아닌 **워드마크 종류**(화이트/
블랙)로 정정. Primary 워드마크는 단색 컬러 배경에 사용 안 한다는 한계
안내를 추가. 명도 사용 구간을 사각지대 없이 재조정.

| 항목 | 정정 전 | 정정 후 |
|---|---|---|
| 명도 바 수 | 3개 (Red / White / Black) | **4개** (Red / Black / White / Gray) |
| Primary (Red) 사용 구간 | 0–30% (밝은 배경) | 0–30% + "단색 컬러 배경에선 사용 안 함" 한계 명시 |
| Black 그라디언트 | 명도 (흰→검정) | **종합** (흰→옅은 노랑→옅은 그레이→중간 그레이) |
| Black 사용 구간 | 0–30% | **0–40%** (밝은~중간 명도, 옅은 컬러 배경) |
| White 그라디언트 | 명도 (흰→검정) | **종합** (흰→IN Lime→MC Blue→BD Purple→KIMES Red→KIMES Black) |
| White 사용 구간 | 70–100% | **40–100%** (중간~어두운 배경, 채도 있는 컬러) |
| Gray 워드마크 | (가이드 누락 — 0–20% 적합 정보 부재) | 4번째 명도 바 추가, 0–20% 명시 |
| 워드마크 stop 수 | 사용 가능 구간 안 3개 | 사용 가능 구간 중앙 **1개** (시각 단순화) |
| 컬러 시뮬레이션 형식 | 그라디언트 바 1개 (이음새) | **칩 5개 + 16px gap** (분리) |

### 근거

### 정정 내용

직전 정정(이 문서 위 항목)의 4-바 그라디언트 시스템도 시각 검증에서
"부드러운 그라디언트가 명도 변화를 단계 시각화하지 못함, 워드마크 1개로는
명도 비교 불가, 사용 X 구간이 텍스트로만 표시되어 차단 신호가 약함"으로
판정. KT BI 가이드(references/kt-ci-guide.pdf p.1)의 11칸 격자 시스템을
정확히 차용해 처음부터 재구현.

KT 시스템과 같은 결의 핵심 차용:

- **11칸 단색 격자** — 0%~100% 10% 단위 균등 분할. 부드러운 그라디언트가
  아닌 명확한 단계.
- **Black/White 워드마크 묶음 처리** — KT가 같은 방식. 각각 별도 바
  대신 한 격자에서 0–40% Black / 50–100% White 묶어 표현.
- **사용 X 구간 빨간 대각선 빗금** — `repeating-linear-gradient(-45deg)`
  로 차단을 시각화. 텍스트 라벨로만 표현하지 않음.

| 항목 | 정정 전 (그라디언트 바) | 정정 후 (KT 11칸 격자) |
|---|---|---|
| 시각 도구 형식 | 부드러운 그라디언트 바 | 11칸 단색 격자 |
| 워드마크 수/바 | 사용 가능 구간 중앙 1개 | 사용 가능 칸마다 1개씩 (총 3~11개) |
| 명도 단계 시각화 | 그라디언트(연속) | 11단계(이산) |
| Black/White 처리 | 별도 바 2개 | 묶음 1개 (KT 방식) |
| 사용 X 표현 | 텍스트 라벨 (`noHint`) | 빨간 대각선 빗금 |
| 컬러 배경 표현 | 그라디언트 바 안에 컬러 stop 혼합 | 별도 격자(컬러 흐름 11칸) |
| 격자 수 | 4 | 4 (Primary / B-W 명도 / B-W 컬러 / Gray) |
| 레이아웃 | 라벨 위·바 아래 세로 | 좌 라벨(200px) + 우 격자, ≤800px 세로 쌓임 |

### 4가지 격자 구조

| # | 라벨 | 규정 | 사용 가능 칸 | 차단 칸 |
|---|---|---|---|---|
| 1 | Primary Wordmark (Red) | 명도 배경색 규정 | 0–30% (4칸) | 40–100% (7칸, 빗금) |
| 2 | Black / White Wordmark | 명도 배경색 규정 ★ 묶음 | 0–40% Black / 50–100% White | 없음 |
| 3 | Black / White Wordmark | 기타 배경색 규정 | 0–4 Black / 5–10 White (KIMES 컬러 흐름) | 없음 |
| 4 | Gray Wordmark | 명도 배경색 규정 | 0–20% (3칸) | 30–100% (8칸, 빗금) |

### 근거

- **부드러운 그라디언트의 시각 어수선함**: 직전 시도의 종합 그라디언트
  (흰→IN Lime→MC Blue→BD Purple→KIMES Red→Black)는 stop 사이가 머지되어
  KIMES 4브랜드 컬러가 분명히 보이지 않음. 11칸 단색 격자로 바꾸면 4브랜드
  컬러가 각 칸에 또렷하게 자리잡음.
- **워드마크 1개로 비교 불가**: OK 구간 중앙 1개만 보여주면 사용자가
  "이 명도에서 이 워드마크가 잘 보이는가"를 다른 명도와 비교 불가.
  11칸에 워드마크 11개를 깔면 명도 변화에 따른 가독성을 한눈에 비교.
- **사용 X 차단 신호 강화**: `noHint` 텍스트 라벨만으론 시각 신호가 약함.
  KT 가이드처럼 빨간 대각선 빗금을 칸 위에 직접 그리면 차단이 즉시 인식.
- **Black/White 묶음의 정보 압축**: 별도 바 2개로 분리하면 페이지 길이
  증가 + 두 워드마크 간 명도 boundary가 시각적으로 안 연결. KT처럼 한
  격자에 묶으면 0–40% Black / 50–100% White boundary가 명료.

## 2026-05-08 — §6.2.5 단순화 (격자 4개 → 2개, 사용 룰 엄격화)

### 정정 내용

직전의 KT 11칸 격자 4개 시스템도 시각 검증에서 "44개 워드마크가 페이지를
가득 채워 답답, Bar 3 일반 컬러 그라디언트가 4브랜드 칩과 정보 중복,
Primary/Black 사용 룰이 KT 가이드에 비해 느슨"으로 판정. 격자 2개로
압축하고 사용 룰을 엄격화 + 워드마크 사이즈 축소.

| 항목 | 정정 전 | 정정 후 |
|---|---|---|
| 격자 수 | 4개 | **2개** (Primary / B-W 묶음) |
| Primary (Red) 사용 구간 | 0–30% (4칸) | **0–10%** (2칸, 매우 옅은 배경 한정) |
| Black 사용 구간 | 0–40% (5칸) | **0–30%** (4칸) |
| White 사용 구간 | 50–100% (6칸) | **40–100%** (7칸) |
| Bar 3 (B-W 컬러 그라디언트) | KIMES 11색 흐름 격자 | **제거** (4브랜드 칩과 중복) |
| Gray 워드마크 격자 | 0–20% 사용 격자(3칸) | **제거** → 한 줄 안내 |
| 격자 안 워드마크 height | 13px (칸 폭의 약 80%) | **9px** (칸 폭의 약 50–60%) |
| 시각 도구 수 | 5 (격자 4 + 칩 1) | **3** (격자 2 + 칩 1 + Gray 안내 1줄) |
| 페이지 안 워드마크 수 | 44개 (격자) + 5개 (칩) = 49 | 13개 (격자) + 5개 (칩) = 18 |

### 근거

- **44 워드마크 → 13 워드마크**: 직전 격자 4개에 칸당 워드마크 1개씩,
  총 44개 워드마크가 페이지를 가득 채워 시각 답답함. 격자 2개로 축소
  하면 13개로 줄고, 워드마크 height를 13→9px로 더 축소하면 칸 안 여백
  도 충분해 시각 가벼움.
- **Bar 3 (B-W 컬러 그라디언트) 정보 중복**: 일반 컬러 11칸 그라디언트
  (흰→옅은 그레이→옅은 라임→IN Lime→...→KIMES Black)가 시각적으로
  KIMES 4브랜드 칩과 같은 정보(어두운 채도 = White / 밝은 채도 = Black)
  를 전달. 참가업체가 진짜 마주치는 케이스는 KIMES 4브랜드 단색 배경이지
  일반 컬러 그라디언트가 아님. 칩으로 충분.
- **Primary 0–30% → 0–10%**: KT 가이드의 Primary 워드마크 사용 룰이
  더 엄격(흰색·매우 옅은 배경에 한정). KIMES Red(#E60012)도 명도 20%
  이상 회색에선 대비 부족. 더 엄격한 룰로 정정.
- **Black 0–40% → 0–30%**: 명도 40% 회색(#999999)에서 Black 워드마크
  대비가 약함. 30%(#B3B3B3)까지로 축소하고 40% 이상은 White가 더 안전.
- **Gray 워드마크 격자 제거**: Gray는 톤다운 자유 사용 케이스라 별도
  격자로 사용 룰을 명시할 정보 양이 부족. 한 줄 안내(칩 아래)로 충분.

### 적용 범위

- components/LogoPage.jsx —
  - `COLOR_FLOW_HEX` 제거 (Bar 3 데이터 불필요).
  - `GRID_BARS` 4개 → 2개 (Primary / B-W brightness만).
  - Primary cells: 0–1만 wm, 2–10 blocked.
  - B-W brightness cells: 0–3 Black, 4–10 White (boundary 30/40).
  - `<InlineLogo height={13} />` → `height={9}` (격자 셀 안).
  - `<p className="lg-gray-note">` 한 줄 안내 추가 (칩 다음).
- components/logo.css —
  - `.lg-gray-note` 신규 (caption size, secondary color).
  - 모바일 svg scale 0.85 → 0.9 (워드마크 작아져 살짝 키움).
- KIMES_BI_Guide_Spec.md §6.2.5 — 격자 2개 구조 + Primary/Black 룰 엄격화
  + Gray 한 줄 안내 + 워드마크 height 명시.

## 2026-05-08 — §8 spec 갱신 (구조 재편 + 톤 정제 + §22.11 신설)

### 정정 내용

### 근거

- **§8.4 "표준" 단어 제거**: "표준 + 권장" 모순. 표준은 강제, 권장은
  비강제. KIMES는 §22 톤 룰(강제 단어 회피, 자유로운 가이드 정체성)에
  맞춰 "권장 조판"으로 정정.
- **§8.5 9 → 7 압축**: "산업 디자인 톤"이 "차분한 톤"과 결 중복, "큰
  디스플레이 톤"이 "광고·헤드라인"과 결 중복. 9개는 시각 부담, 6개는
  빈약, 7개가 페이지 무게 적정선.
- **디자이너 언어 회피 (§22.11)**: 일반 사용자(참가업체 마케팅·홍보
  담당자)는 폰트 분류(명조체/산세리프/Serif), 디자인 톤 분류(미니멀/
  임팩트/디스플레이) 같은 디자이너 어휘를 즉시 인식 못 함. "어떤
  회사·매체·상황에 적합한지"로 라벨링하면 사례를 보고 자기 자료에
  바로 매핑 가능.
- **카드에서 라벨 제거**: 라벨이 사례를 미리 분류해버리면 사용자가
  시각을 보기 전에 분류된 인상으로 받아들임. 시각 + 사용 상황 한 줄
  구조로 "이 모양은 이런 상황에 잘 맞는다"는 직접적 매핑.
